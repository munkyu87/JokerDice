# RogueRoll MVP 구조 문서

## 목표

이 MVP는 발라트로 감성의 핵심인 `한 턴의 계산 재미`, `빌드가 바뀌는 성장`, `제약을 비트는 보스전`을
주사위 기반으로 옮긴 첫 구현입니다.

이번 버전에서 반영한 핵심 방향은 아래와 같습니다.

- 족보 단독 점수가 아니라 `족보 점수 + 주사위 숫자 합` 구조 사용
- 액션 카드를 일회성 소모품이 아니라 `드로우/버림/덱 압축`이 있는 덱 순환 구조로 변경
- 조커를 `트리거 기반`으로 설계해서 확장성과 충돌 처리를 쉽게 만듦
- 보스를 완전 봉쇄형이 아니라 `플레이 방식을 비트는 제약형`으로 설계
- 모바일 세로형 화면에서 한눈에 현재 점수, 리소스, 주사위, 카드, 조커를 볼 수 있게 구성

## 현재 구현 범위

### 게임 루프

1. 스테이지 시작
2. Hand 시작 시 주사위 5개 자동 롤
3. 기본 손패 3장 드로우
4. 리롤 또는 액션 카드 사용
5. 점수 확정
6. 목표 점수 도달 시 보상 선택
7. 상점 진입
8. 다음 스테이지 진행

현재는 `Ante 1~8` 구조이며, 각 엔티는 `Small Blind -> Big Blind -> Boss Blind` 3스테이지로 진행됩니다.
`Ante 8 Boss Blind`를 클리어하면 이번 런을 완전히 클리어한 것으로 처리합니다.

## 점수 시스템

공식은 다음과 같습니다.

```ts
finalScore = (handBase + diceBase + bonusBase) * multiplier;
```

### 의도

- `handBase`: 족보의 구조적 가치
- `diceBase`: 숫자 조작 카드와 고숫자 전략의 가치
- `bonusBase`: 조커/보스가 주는 평면 보너스
- `multiplier`: 후반 폭발력을 만드는 핵심 축

이 구조 덕분에 같은 `Pair`라도 낮은 숫자와 높은 숫자의 체감 차이가 살아납니다.

## 족보

현재 구현된 족보:

- `high_card`
- `pair`
- `two_pair`
- `three`
- `straight`
- `full_house`
- `four`
- `five`

## 카드 구조

카드는 모두 데이터 기반이며, 현재는 다음 5종이 구현되어 있습니다.

- `Precision Reroll`
- `Loaded Six`
- `Raise All`
- `Mirror High`
- `Even Polish`

### 덱 순환 규칙

- Hand 시작 시 기본 3장 드로우
- 조커 효과로 Hand별 드로우 장수가 늘어날 수 있음
- 카드 사용 시 버림 더미로 이동
- 사용하지 않은 손패도 Hand 종료 시 버림 더미로 이동
- 드로우 더미가 비면 버림 더미를 섞어서 재사용
- 보상/상점으로 카드 추가 가능
- `Deck Trim`으로 카드 제거 가능
- 일부 조커는 Hand 시작 시 `손패 교체` 자원을 제공하며, 현재 손패를 버리고 같은 수만큼 새로 뽑을 수 있음

즉, 이번 버전부터 이미 덱빌딩 감각의 핵심인 `압축`, `순환`, `중복 가치`를 확인할 수 있습니다.

## 조커 구조

조커는 아래 트리거를 기반으로 동작합니다.

```ts
type JokerTrigger = 'onHandStart' | 'beforeScore' | 'afterScore';
```

### 현재 구현된 조커 예시

- `Lucky Reroll`: Hand 시작 시 무료 리롤 +1
- `Full Grip`: Hand 시작 시 손패 +1
- `Fresh Deal`: Hand 시작 시 손패 교체 +1
- `Six Master`: 6 개수만큼 배수 증가
- `Even Power`: 모두 짝수면 배수 증가
- `Triple Boost`: 트리플 이상 족보면 배수 증가
- `Straight Spark`: 스트레이트 보너스
- `Steady Pair`: 페어 계열 보너스
- `Golden Touch`: 고득점 시 골드 획득

### 조커 적용 순서

- 화면에 보이는 순서대로 왼쪽에서 오른쪽으로 적용
- 보스가 조커를 막는 경우, 오른쪽 슬롯부터 비활성화

이 순서를 명시해둔 이유는 이후 조커가 많아질수록 밸런싱과 디버깅의 기준점이 되기 때문입니다.

## 보스 구조

보스는 빌드를 아예 죽이지 않고 계산 방식을 틀어버리는 제약으로 설계했습니다.

현재 구현된 보스:

- `Static Sleeve`: 가장 오른쪽 조커 1개 비활성화
- `Ceiling Jam`: 5와 6을 점수 계산 시 4로 낮춤
- `Odd Tax`: 홀수 주사위마다 기본 점수 차감

## 빌드 축

현재 데이터는 아래 빌드 축을 염두에 두고 태그를 가지고 있습니다.

- `high`: 고숫자 / 6 시너지
- `even`: 짝수 빌드
- `set`: 페어, 트리플, 포카드 계열
- `sequence`: 스트레이트 계열
- `economy`: 골드 수급
- `consistency`: 안정성, 리롤, 보정

이 태그는 이후 보상 풀 제어, 등장 확률, UI 필터, 메타 진행으로 확장하기 좋습니다.

## 파일 구조

```text
App.tsx
docs/
  rogueroll-mvp.md
src/
  game/
    data.ts
    engine.ts
    types.ts
    useRogueRollGame.ts
  screens/
    RogueRollScreen.tsx
```

### 파일 역할

- `src/game/types.ts`
  - 게임 전반 타입 정의
  - 카드, 조커, 보스, 보상, 상점, 점수 결과 타입 관리

- `src/game/data.ts`
  - 카드 / 조커 / 보스 / 스테이지 데이터 정의
  - 데이터 추가만으로 콘텐츠 확장 가능

- `src/game/engine.ts`
  - 주사위 롤
  - 족보 판정
  - 점수 계산
  - 조커 적용
  - 덱 드로우/버림/제거
  - 보상/상점 샘플 생성

- `src/game/useRogueRollGame.ts`
  - React Native 화면이 사용할 런 상태 관리 훅
  - 단계 전환, 리롤, 카드 사용, 점수 확정, 보상/상점 흐름 담당

- `src/screens/RogueRollScreen.tsx`
  - 세로형 모바일 UI
  - 현재 점수, 보스, 주사위, 카드, 조커, 상점/보상 상태를 한 화면에서 확인 가능

## 의도적으로 남겨둔 다음 단계

현재는 구조가 명확한 MVP를 우선했기 때문에 아래는 다음 확장 포인트입니다.

- 엔티별 목표 점수/보상 밸런스 세부 조정
- 런 시작 전 스타터 주사위 선택과 색상별 챌린지 규칙 추가
- 조커 희귀도 및 등장 가중치
- 카드 드로우량, 손패 교체, 조커 슬롯 확장
- 카드/조커 설명을 로컬라이징 가능한 문자열 키 구조로 분리
- 보상 풀에서 현재 빌드 태그 기반 추천 강화
- 애니메이션, 사운드, 피드백 연출
- 세이브/불러오기

## 구현 원칙 요약

- 규칙은 단순하게 유지
- 조커는 강하게, 대신 발동 타이밍은 명확하게
- 보스는 불합리한 봉쇄보다 플레이 스타일 변형 중심
- 수치보다 구조를 먼저 단단하게
- UI보다 코어 엔진을 먼저 분리해서 이후 확장 비용을 낮춤

## UI 방향 업데이트

이번 단계부터 메인 플레이 화면은 `스크롤 없는 1화면 전투판`을 기준으로 잡습니다.

### 핵심 원칙

- 전투 중 필요한 정보는 모두 메인 화면에 상시 노출
- 설명성 문구는 시작 화면 또는 가이드 화면으로 분리
- 족보표, 보스 상세, 최근 계산 로그는 팝업으로 분리
- 보상, 상점, 카드 제거도 전체 화면 전환 대신 오버레이 모달로 처리

### 현재 배치 의도

```text
[상단]
현재 점수 / 목표 점수 / 예상 족보 / 메시지
Stage / Hands / Rolls / Gold / Boss

[중상단]
주사위 5개
현재 계산식과 주요 보정 노트

[중하단]
조커 슬롯 5개

[하단]
가변 손패 카드열

[맨 아래 고정]
전체 선택 / 리롤 / 점수 확정
```

### 카드 인터랙션 방향

- 메인 화면에서는 카드 설명을 항상 노출하지 않음
- 손패는 추후 카드 아트가 들어갈 자리를 고려해 `전면 카드`처럼 간단히 노출
- 카드 탭은 정보 확인, 실제 사용은 `드래그 앤 드롭`으로 분리해서 오조작을 줄임
- 드래그 중에는 카드 패널 상단에 `사용 드롭 존`을 띄워 넣는 감각을 살림
- 카드를 탭하면 해당 카드 위에 설명 팝업을 띄움
- 카드를 길게 누르면 사용

즉, 평소에는 직관적인 카드 면만 보이고, 자세한 텍스트는 필요할 때만 확인하는 방향입니다.

### 조커 인터랙션 방향

- 조커도 항상 설명문을 노출하지 않음
- 기본 화면에서는 작은 슬롯형 카드처럼 보이게 유지
- 조커를 누르면 이름, 효과, 발동 타이밍을 별도 설명 박스로 노출
- 즉시 읽어야 하는 정보는 줄이고, 필요한 순간만 상세 확인 가능하게 설계

### 비주얼 톤 방향

- 전체 배경은 밝은 앱 화면보다 `어두운 게임 보드`에 가깝게 유지
- 액션 카드는 청색 계열을 기본으로 하되 태그에 따라 프레임 색상을 다르게 부여
- 조커는 더 강한 대비와 진한 색 프레임을 사용해 특별 슬롯처럼 보이게 설계
- 카드와 조커 모두 추후 실제 아트가 들어와도 교체하기 쉬운 프레임형 레이아웃을 우선 사용

### 왜 이런 구조로 가는가

- 작은 화면에서도 전투 중 시선 이동이 짧아짐
- 발라트로처럼 "현재 상태를 한 번에 읽는" 리듬을 만들 수 있음
- 정보량은 유지하면서도, 상세 설명은 팝업으로 숨겨 과밀감을 줄일 수 있음

### 이후 UI 확장 포인트

- 조커/카드에 실제 카드형 아트 프레임 적용
- 주사위 롤 애니메이션과 선택 하이라이트 강화
- 족보 정보 팝업을 카드 도감식 레이아웃으로 확장
- 보스 오버레이에 경고 연출 추가
